# Skill: Deploy and Push

이 Skill은 프로젝트의 변경 사항을 안전하게 원격 저장소에 반영하고 실 서비스(Vercel)에 배포하는 자동화 가이드를 제공합니다.

## 사용 목적
- 개발 완료 후 수동 입력 번거로움 해소
- 커밋, 푸시, 배포 과정을 누락 없이 한 번에 수행

## 실행 지침
1. 작업 디렉토리가 `mind-weather`인지 확인합니다.
2. 현재 변경된 파일이 있는지 `git status`로 점검합니다.
3. 다음 순서로 명령을 실행합니다:
   - `git add .`
   - `git commit -m "[변경사항 요약]"`
   - `git push`
   - `vercel --prod`
4. 배포 완료 후 제공되는 Vercel URL을 통해 정상 작동 여부를 확인합니다.

## 예시 프롬프트
"지금까지 한 작업들 다 깃에 올리고 버셀 배포해줘"
-> 이 요청을 받으면 본 Skill 지침에 따라 `/deploy` 워크플로우를 실행하거나 직접 명령어를 수행합니다.
