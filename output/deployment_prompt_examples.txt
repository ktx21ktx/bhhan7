[Mind Weather] 배포 자동화 활용 프롬프트 예시 가이드

생성된 'deploy-and-push' Skill 및 '/deploy' 워크플로우를 통해 깃 푸시와 Vercel 배포를 한 번에 자동화할 수 있습니다. 아래 예시들을 복사해서 사용하세요.

---

1. 가장 빠르고 간결한 요청 (Slash Command)
   명령어 입력창에 아래만 입력하면 즉시 전체 배포가 시작됩니다.
   - /deploy

2. 자연스러운 대화형 요청
   작업 수정 후 AI에게 편하게 지시할 때 사용합니다.
   - "지금까지 작업한 내용들 전부 깃허브에 올리고 실제 서버(버셀)에도 배포해줘."

3. 특정 커밋 메시지를 지정하고 싶을 때
   변경 기록을 프로젝트 관리 용도로 남길 때 유용합니다.
   - "커밋 메시지는 '달력 아이콘 수정 및 DB 최적화 완료'라고 해서 푸시하고 배포까지 끝내줘."

4. 작업 확인 후 연달아 요청할 때
   - "방금 수정한 코드 잘 작동하는 것 같네. 이제 /deploy 워크플로우 실행해서 라이브 서버에 반영해줘."
